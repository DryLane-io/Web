World-model experiment · 8 runs

When does structure beat
a learned world model?

A physics-grounded predictor versus a black-box neural net, on the same 2D world. Eight experiments, run to break the idea, not to flatter it. One honest finding: structure wins exactly where the world is data-poor, fast, and changing — and a strong baseline holds everywhere else.

Live · lead the target

Both models aim by predicting the future

Each model must guess where the target will be, then fire a straight shot to meet it. Both were trained only on slow targets — the teal band on the slider. Push the speed past that band and the black-box aims short: it can't imagine motion faster than it has seen. The structured model just integrates the physics, so speed is no obstacle.

The scorecard

Three wins, two honest limits

The wins aren't a grab-bag. Scarce data, sudden novelty, a shifting world: that's the description of an agent living in reality without a data lake or a supercomputer.

StructureLittle data. Reaches oracle accuracy from ~8 samples; the black-box needs hundreds.
StructureFast reaction to novelty. ~9× more accurate intercepting a never-seen fast target with a split second to react.

Instrument · non-stationary world

Adapting when the wind changes

The world is calm, then the wind shifts twice. After each shift both models spike identically — neither saw it coming. Then they learn online from what they observe. A wind change alters only the force, so structure re-learns a tiny correction and keeps its physics; the black-box must repair its whole entangled operator, and plateaus.

Kept honest

Where the black-box holds

Given a rich, on-distribution dataset and a long horizon, a black-box that has learned the exact operator is genuinely hard to beat — it matched or beat the learned structured model in those regimes here (structure's oracle ceiling, with the true force, still led). And structure only helps if its form is right: give it the wrong physics and it diverges worse than the black-box. The claim is narrow on purpose. It isn't "structure always wins" — it's "structure wins in the data-poor, novel, non-stationary corner," and that corner happens to be where real embodied agents live.

Method

How it was run

A 2D world with gravity + quadratic drag. Structured model = a fixed kinematic integrator with a small learned force model plugged in (state → acceleration); it gets the physics form for free and only learns the unknown force. Black-box = a plain neural net mapping state → next state, no physics. Same data, same budget. Everything is a self-contained C# console program with a hand-rolled MLP and no dependencies; each experiment is a table the program prints. The demo above reproduces the out-of-distribution mechanism faithfully in the browser: the black-box cannot emit a displacement larger than it trained on, so it under-predicts fast motion — exactly the failure measured in the runs.
